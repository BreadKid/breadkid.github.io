Java基础

基本数据类型：int、short、long、float、double、char、boolean、byte

==与equals（==）：

  • 基本类型比较值
  • 引用类型比较引用地址
  • 重写equals方法会把比较地址改为比较值

对象的hashcode相同，值不一定相同

String str=”i”：常量池中
String str=new String(“i”)：对内存中

final：类不可被继承，方法不可被重写，变量为常量不可被修改
finally：异常处理始终会执行的代码块，catch可以省略
finalize：垃圾回收时调用

String：不可变
StringBuffer：可变，线程安全，性能差些
String Builder：可变，现成不安全，性能更好

抽象类：不一定非要有抽象方法，不能直接实例化，单继承，有构造函数，有并能运行main函数
普通类：不能包含抽象方法，可以直接实例化
接口：不能有抽象方法，没有构造函数，没有main函数，多实现

IO流：输入输出流，字节字符流

BIO：同步阻塞IO，单线程场景
NIO：同步非阻塞IO，BIO的升级，基于channel通讯
AIO：异步非阻塞IO，NIO的升级，基于事件与回调机制

常用类库：

集合

java.util.Collection：集合接口
java.util.Collections：集合工具类

时间复杂度：值越大，效率越低
  • O(1): 依次执行
  • O(n): 单次for循环
  • O(logN): 单次while循环
  • O(NlogN): 单次for循环嵌套while循环
  • O(n^k): k层for循环
List

Array：指定大小，允许基本类型和对象
ArrayList:数组，支持随机访问，O(1)，异步
LinkedList:双向循环链表，O(n)
Vector：同步，线程安全。

Map

HashMap:单线程非同步，无序，允许空键值，增删快，数组O(1)->链表8 O(n)->红黑树O(logn)，线程不安全
Hashtable:同步，不允许空键值，线程安全，作用==Collections.synchronizedMap(HashMap)
TreeMap：适用于有序key遍历
ConcurrentHashMap：多线程，线程安全

Set

HashSet：基于HashMap

集合在多线程场景下替代：

  • vector
  • Collections.synchronized*
  • juc.CopyOnWriteArray*
  • HashTable

线程

进程：程序运行与资源分配的基本单位，有独立内存单元
线程：进程的实体，进行cpu调度，共享进程的内存
守护线程：为其他线程提供服务的线程
创建线程：

  • extends Thread
  • implements Runnable，实现void run()
  • implements Callable，实现T call()，回调

线程状态：创建、就绪、运行、阻塞、死亡

sleep(): 未释放锁
wait(): 释放锁，需要被唤醒notify

线程池

volatile：

  • 锁定内存，
  • 修饰变量
  • 不会造成线程阻塞

synchronize：

  • 锁定当前变量，只有当前线程可访问，其他线程阻塞
  • 修饰变量，方法，类
  • 会造成线程阻塞
  • 无法判断获取锁
  • 会自动释放锁

Lock：

  • 可判断是否获取锁
  • 不会自动释放锁，unlock()

反射

Java运行时，反射机制能够在运行时：

  • 判断对象所属的类
  • 构造类的对象
  • 判断类的成员变量与方法
  • 调用对象的方法

动态代理

场景：AOP，事物，权限，日志等

序列化

场景：

  • 将对象写入文件/DB
  • 用套接字进行网络传输
  • 通过rmi传输对象

JVM

浅拷贝：栈中复制创建新应用地址，指向同一个堆内存空间

数据库

三大范式：字段不冗余，表必有主键，外键关联主键

ACID是关系型数据库的基本属性

防止sql注入：

  • 字符串过滤
  • PreparedStatement
  • mybatis不使用${xxx}，换成#{xxx}

MySQL(InnoDB)

基本数据类型：
只有引擎InnoDB支持事物，支持外键，使用索引组织表（辅助索引为key，聚簇索引为value），底层B+树

索引表在内存中，读写快

  • 普通索引、INDEX
  • 唯一索引、允许null
  • 主键索引、
  • 组合索引、遵循最左前缀集合，列值的组合必须唯一
  • 全文索引，只有char、varchar，text 类型可建

聚簇索引：

  • 主键
  • 没有主键，选择第一个非空的唯一索引
  • 没有主键和唯一索引，会隐式定义一个6字节的 rowid 主键

事务隔离级别：

  • 读未提交
  • 读已提交
  • 可重复读：MVCC机制（多版本并发控制），可以使用快照读，而不会被阻塞
  • 可串行化

事务隔离级别原理：

脏读：读到未commit的update数据，只在读未提交隔离级别出现
幻读：读到未commit的insert数据，幻读在读未提交、读已提交、可重复读隔离级别都可能会出现

行级锁：

  • 对于两个事务处理同一行数据的场景，实现在索引上
  • 减少锁冲突，并发更高
  • 占用更多内存，会有死锁

死锁：主动死锁检测策略：主动回滚死锁链条中代价更低的事务(持有锁较少的事务)，让其他事物继续执行
